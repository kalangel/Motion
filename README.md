# Prestige Estate — сайт риелтора с 3D-туром по дому

Премиальный лендинг для продажи загородного дома с интерактивным виртуальным туром:
клиент «ходит» по дому прямо в браузере, приближает детали, вылетает за окно и видит
всё строение целиком.

## Возможности тура

| Режим | Что делает | Как включить |
|---|---|---|
| 🚶 Прогулка | Ходьба от первого лица: клик по полу — шаг, лестница ведёт на 2 этаж | кнопка «Прогулка» или клик по комнате |
| 🔍 Зум | Приближение/отдаление деталей | колесо мыши, кнопки +/−, щипок на телефоне |
| 🪟 Вылет за окно | Отдалите до упора — камера вылетает через ближайшее окно и показывает весь дом | колесо «в минус» до заполнения индикатора |
| 🏡 Весь дом | Орбитальный облёт здания и участка | кнопка «Весь дом» |
| ✂️ Разрез | Дом без крыши — планировка сверху (dollhouse) | кнопка «Разрез» |
| 📷 360° фото | Реальная панорамная фотография, снятая на объекте | кнопка «360° фото» или маркер в кабинете |

Работает на компьютере, планшете и телефоне. Зависимостей у сборки нет —
это статический сайт (Three.js лежит в `vendor/`).

## Запуск локально

```bash
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

Любой статический сервер подойдёт. Открывать `index.html` двойным кликом тоже можно,
но браузер может блокировать загрузку текстур с `file://` — сервер надёжнее.

## Деплой

Статический хостинг без сборки:

- **Vercel**: Import репозитория → Framework Preset: **Other** → Build Command: пусто →
  Output Directory: `./` → Deploy.
- **GitHub Pages**: Settings → Pages → Deploy from a branch → выбрать ветку и `/ (root)`.
- **Netlify**: перетащить папку в интерфейс или подключить репозиторий (без команды сборки).

## Структура

```
index.html        — лендинг (шапка, тур, характеристики, галерея, контакты)
css/style.css     — стили
js/house.js       — процедурная 3D-модель дома, участка и мебели
js/tour.js        — движок тура: режимы, камера, перелёты, панорама
js/main.js        — связка интерфейса с движком
assets/pano.jpg   — реальная 360°-панорама (equirectangular)
assets/gallery/   — кадры для галереи
vendor/three.min.js — Three.js r128
```

## Как адаптировать под свой объект

1. **Тексты и цена** — правятся прямо в `index.html` (название, цена, площадь,
   телефон и почта агента).
2. **360°-панорамы** — замените `assets/pano.jpg` на свою equirectangular-панораму
   (соотношение сторон 2:1). Точка съёмки настраивается в `js/tour.js`
   (`tour.panoRoom`), ориентация — `panoSphere.rotation.y`.
3. **Планировка** — комнаты, окна и мебель описаны в `js/house.js`:
   - `rooms` — имя, этаж, точка входа (`pos`), направление взгляда (`look`)
     и окно, через которое камера влетает в комнату (`win`);
   - `wallRun(...)` — стены с проёмами, `windowUnit(...)` — окна;
   - мебель — простые функции-конструкторы (`sofaUnit`, `bedUnit`, `wardrobe`…).
4. **Галерея** — положите свои кадры в `assets/gallery/` (имена файлов в `index.html`).

## Технические детали

- Three.js r128 (UMD), без сборщика — файлы можно менять и сразу обновлять страницу.
- Освещение: hemisphere + солнце с тенями (PCF) + точечные светильники в комнатах;
  материалы Phong (per-pixel), т.к. Lambert в r128 считает точечный свет по вершинам.
- Перелёты камеры (в окно/из окна) — сплайны CatmullRom с плавным поворотом взгляда;
  на время пролёта стёкла становятся прозрачными.
- Переход между этажами — маршрут через лестницу, между комнатами — через дверные проёмы.
